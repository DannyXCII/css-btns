Standard Buttons

Our standard coloured buttons. The default button is grey and you can change the colour easily with our no-nonsense class names. Not happy with the colours that we've chosen? No problem! We've made it really easy to change the default settings with just one little line of code, by including SCSS variables.

Why not add your own colours and extend upon this library? BTNS has been built with extendability in mind and it's been designed to be as easy to follow as possible. With under 150 lines of code*, BTNS is a doddle to get to grips with.

Not familiar with SCSS? Check out our handy quick start guide to help you get started! Alternatively BTNS is also available in pure CSS - albeit making customisation more difficult.

* Comments not included.

Quick Start Guide

SASS/SCSS BTNS:

This guide is currently good for Windows users. If you are using an alternative operating system please check back later for more guides. Want to help out by creating a guide for other operating systems? Email me to get involved!

If you are unfamiliar with SASS then do not fear, this handy guide should help you to get started using SCSS BTNS. We'll walk you through the process as simply as possible.

After downloading and extracting the 'sass-btns' folder, you will notice two files: btns.scss and _settings.scss. Copy both of these into a new folder in your project called SCSS (or whatever you want to call it really). Check out the screenshot below to see an example of a structured project including BTNS.

You'll notice that right now, the HTML document is linking to a non-existent CSS file! That's ok though, we'll get round to creating that in just a moment (magically). The next step is to setup SASS on your machine. First of all, download the Ruby Installer and run the program.

Once Ruby is installed on your device, open up the command window and type gem install sass. This should kick off the installation process for SASS. Once this is done, you are now able to use SASS on your machine, hooray!

The next step involves your command line once again. Once installation has completed type cd (yes, cd followed by a space - it stands for change directory), then target your project folder (New Buttons Project in our case). The easiest way to do this is to open your command line and file explorer side by side, then drag your project folder into the command line - then press enter.

Next, type sass --watch scss:css on the next line (after hitting return). This will watch all the files in 'New Buttons Project/scss' and will generate a new stylesheet in the css folder. Your command line should look something like this:

Now you can navigate back to your project and if you've followed all of these steps, you should now see an auto generated btns.css file inside your CSS folder - how cool is that? The generated btns.css file is the file that needs to be uploaded to your website, the SCSS files stay on your mcahine for local editing!
